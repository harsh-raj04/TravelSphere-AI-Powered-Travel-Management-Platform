# TravelSphere Phase 1 Wireframe Checklist

## 1. Goal
Prepare low-fidelity wireframes for all role-based MVP flows before implementation.

---

## 2. Customer UI Screens
- [ ] Login screen
- [ ] Register screen
- [ ] Package listing screen
- [ ] Package detail screen
- [ ] Booking form screen
- [ ] My bookings screen
- [ ] Booking detail/status screen

## 3. Agent UI Screens
- [ ] Agent login screen
- [ ] Agent dashboard screen
- [ ] Create package screen
- [ ] Edit package screen
- [ ] My packages list screen
- [ ] Package bookings management screen

## 4. Admin Panel Screens
- [ ] Admin login screen
- [ ] Admin dashboard overview
- [ ] All bookings table
- [ ] Booking detail drill-down
- [ ] Basic analytics widgets (bookings, revenue, top agents, top packages)

---

## 5. Mandatory Flow Maps
- [ ] Customer booking flow (browse -> details -> booking -> status)
- [ ] Agent publish flow (create package -> list -> bookings)
- [ ] Admin tracking flow (bookings list -> detail -> analytics)

---

## 6. UI Standards to Lock in Phase 1
- [ ] Shared navigation pattern per role
- [ ] Form validation message pattern
- [ ] Table and filter UX pattern
- [ ] Status badges pattern (pending/confirmed/cancelled/completed)
- [ ] Empty and error state components

---

## 7. Output Format
Wireframes can be created in Figma, Excalidraw, or Draw.io.

Required export set:
- [ ] One PDF or image set grouped by role
- [ ] One short note explaining each core user flow

---

## 8. Completion Rule
Wireframe checklist is complete only when all three role flows can be understood by a developer without verbal explanation.
